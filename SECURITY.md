# Security Policy

## Supported Versions

Only the **latest tagged release** is supported with security updates.

| Version | Supported |
|--------|-----------|
| Latest release | ✅ |
| Older releases | ❌ |

---

## Reporting a Vulnerability

If you discover a security issue, **do not open a public issue**.

### How to report
Send a private report with:
- description of the issue
- steps to reproduce
- potential impact
- affected version

to:

**Email:** security@yourdomain.example  
*(replace with a real address)*

You may also use GitHub private security advisories if enabled.

---

## What counts as a security issue
- Authentication or authorization bypass
- Data leakage (scores, backups, tokens)
- Backup or encryption weaknesses
- Remote code execution
- Privilege escalation in Discord

## What does NOT count
- Discord permission misconfiguration
- User error during setup
- Missing optional hardening (documented)

---

## Response process
1. Acknowledgement within **7 days**
2. Fix or mitigation planned
3. Coordinated disclosure after release

We prefer **boring, responsible disclosure**.
